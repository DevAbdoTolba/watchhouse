<objective>
Deliver the last two Phase 1 behaviors:
1. **Main-stream on demand with DVR-wide serialization** (ING-07 + ING-08) — a single `MainStreamGrabber` thread owning a global `threading.Semaphore(1)` and a 2-s TTL cache, exposing a synchronous `grab_main_frame(camera_id) -> bytes | None`. Phase 3's `TriggerEvaluator` will call this directly.

A new `--live` CLI flag on `python -m home_cctv` runs the whole thing against `cameras.yaml`.

Purpose:
- ING-07: Main-stream captures opened on demand for a single frame on trigger, then released — never held open continuously — with a short (~2 s) TTL cache for burst triggers on the same camera.
- ING-08: Concurrent main-stream opens are serialized via a single global `threading.Semaphore(1)` so the DVR's 6-session cap (PHASE0-REPORT: tested up to 6 OK) is never exceeded with 4 subs + 1 main + phone app = 6.
- G-02 start-degraded + G-03 synchronous API + Phase 0 `ShutdownSupervisor` SIGINT drain.

   - `def grab_main_frame(self, camera_id: int) -> Optional[bytes]`:
     - **First cache check** (pre-semaphore, fast path): under `self._cache_lock`, check `self._cache.get(camera_id)`. If entry exists and `time.monotonic() - entry.cached_at_monotonic < self.ttl_s`, log `main_grab_cache_hit camera_id=%s age_s=%.2f` and return `entry.jpeg_bytes`.
     - Otherwise: acquire the semaphore with a timeout that respects `stop_event`:
       ```python
       while not self.stop_event.is_set():
           if self.semaphore.acquire(timeout=0.5):
               break
       else:
           return None
       ```
     - **Second cache check (double-checked locking — ING-07 invariant):** immediately after acquiring the semaphore and BEFORE calling `_open_read_release`, re-check under `self._cache_lock`. A second concurrent caller that queued behind the semaphore while the first caller was executing `open → read → release` MUST NOT re-open the capture; it should see the fresh cache entry placed by the first caller. If the re-check finds a still-fresh entry: log `main_grab_cache_hit_after_wait camera_id=%s age_s=%.2f`, **release the semaphore**, and return `entry.jpeg_bytes`. This is the critical path that enforces "exactly one short-lived open under burst" (ING-07).
     - Only on second-check miss: call `self._open_read_release(camera_id)` which returns `Optional[bytes]`.
     - On success: populate cache under `self._cache_lock`.
     - Always release the semaphore in `finally` (but not double-release — only release it in `finally` if the second-check path didn't already release it; cleanest implementation is a `semaphore_released` local flag).
     - Return the bytes or None.
   - `def _open_read_release(self, camera_id: int) -> Optional[bytes]`:
     - Resolve url = `self.url_resolver(camera_id)`.
     - Log `main_grab_open camera_id=%s` (URL never interpolated — credential-leak guard).
     - `cap = self._capture_factory(url)`.
     - If `not cap.isOpened()`: log `main_grab_open_failed camera_id=%s`; return `None`.
     - Try up to `MAIN_STREAM_MAX_READ_ATTEMPTS` times: `ok, frame = cap.read()`. If ok and frame is not None, break; else continue.
     - If no frame: log `main_grab_read_failed camera_id=%s attempts=%d`; `cap.release()`; return `None`.
     - Encode via `cv2.imencode(".jpg", frame, [int(cv2.IMWRITE_JPEG_QUALITY), MAIN_STREAM_JPEG_QUALITY])`. If `ok2` is False: log, release, return None.
     - `cap.release()` (always, in a `finally`).
     - Log `main_grab_ok camera_id=%s bytes=%d`.
     - Return the JPEG bytes.
